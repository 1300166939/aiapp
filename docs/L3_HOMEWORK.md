# Codex 训练营 L3 作业：Test Knowledge Report Studio

## 1. 原始需求

我当前最需要的 APP 是一个“自动化测试失败诊断报告工作台”。

真实业务背景：

- 我正在做自动化测试平台的测试知识库建设。
- 项目里已经有 `TesttoolPlatformAPI/ApiTestWeb/test_knowledge` 模块。
- 失败诊断服务已经从纯 RAG 调整为“实时数据库事实 + 规则诊断 + RAG 历史经验”。
- 日常排查中经常需要把 `userId is required`、`params err`、`500`、依赖缺失、测试集顺序错误等问题整理成可解释报告。

痛点：

- 手工写诊断报告容易漏证据。
- Codex 直接生成结论容易过度自信。
- 每次都要重复写“哪些能交给 Codex，哪些必须人工判断”。
- 缺少一个能保留“初稿-运行-纠偏-更新-复验”过程的工具。

因此，我做了一个 APP：`Test Knowledge Report Studio`。

## 2. MVP 版本

MVP 聚焦一个闭环：让测试同学在单页内完成一份可提交、可复验、可沉淀的 Codex 诊断报告。

功能包括：

1. 录入失败上下文：项目、模块、环境、caseNum、API Path、失败现象。
2. 维护证据链：证据、来源、支持判断、可信度、是否需要人工确认。
3. 记录闭环：初稿、运行、纠偏、更新、复验。
4. 标注人机边界：Codex 可以处理什么，必须人工判断什么。
5. 导出 Markdown 报告。
6. 本地保存草稿。
7. 质量雷达自动检查是否满足基本验收项。

## 4. 沉淀项目 Skill

Skill 文件：

```text
skills/report-studio/SKILL.md
```

核心规则：

- 先拿证据，再写结论。
- 不能把 LLM 推测当事实。
- 改库、重跑、调整依赖、上线判断必须人工确认。
- 每次诊断都保留初稿、运行、纠偏、更新、复验记录。
